# 项目规则

## 开发铁律：doc 三文档

本项目目录下设置 `doc/` 文件夹（没有就创建），内部维护 3 个文档：

| 文档 | 用途 |
|------|------|
| `doc/ARCHITECTURE.md` | 项目代码工程结构文档 |
| `doc/FEATURES.md` | 项目产品文档 |
| `doc/CHANGELOG.md` | 变更日志，记录自上一次以来的修改内容、修改时间、对应 git 提交信息；可一直追加，不覆盖历史 |

**内容要求**：`ARCHITECTURE.md` 和 `FEATURES.md` 必须精炼但覆盖项目所有信息，便于 AI 阅读后快速理解项目功能与代码结构、上手开发。

## 协作流程铁律

1. 用户提出需求 → 进行开发
2. 开发完成后，等待用户**主动验证**功能正确性
3. **只有在用户明确指示后**，才可以提交代码（git commit/push）和同步文档（更新 `ARCHITECTURE.md` / `FEATURES.md` / `CHANGELOG.md`）
4. 用户没有明确指示时，**严禁**提交代码、修改或增加文档

> 也就是说：每次开发任务收尾时，只做"待验证"状态提示，不主动执行 git 操作；等用户说"可以提交"或类似明确指示，再执行提交 + 文档同步 + CHANGELOG 追加。
